# Extraction Types

VizAPI.ai supports various extraction types, each optimized for specific document formats and use cases. This guide provides details about each extraction type and the data fields they extract.

## Receipt Extraction

The `receipt` extraction type is designed to extract structured data from retail and service receipts.

<Frame>
  <img src="/images/receipt-extraction.png" alt="Receipt Extraction Example" />
</Frame>

### Extracted Fields

| Field | Description | Example |
| ----- | ----------- | ------- |
| `merchant` | Name of the merchant or store | "Walmart" |
| `date` | Date of purchase (ISO format) | "2023-05-15" |
| `time` | Time of purchase (when available) | "14:30" |
| `total` | Total amount | 42.99 |
| `subtotal` | Subtotal amount (before tax) | 35.97 |
| `tax` | Tax amount | 7.02 |
| `currency` | Currency code | "USD" |
| `payment_method` | Method of payment | "VISA" |
| `receipt_number` | Receipt identifier | "R-12345" |
| `items` | Array of purchased items | See below |

Each item in the `items` array contains:

```json
{
  "description": "Office Supplies",
  "quantity": 2,
  "unit_price": 9.99,
  "amount": 19.98
}
```

### Use Cases

- Expense tracking and reimbursement
- Bookkeeping and accounting
- VAT/tax reclaim
- Consumer spending analysis

## Invoice Extraction

The `invoice` extraction type is designed for business invoices and bills.

<Frame>
  <img src="/images/invoice-extraction.png" alt="Invoice Extraction Example" />
</Frame>

### Extracted Fields

| Field | Description | Example |
| ----- | ----------- | ------- |
| `invoice_number` | Invoice identifier | "INV-2023-001" |
| `date` | Invoice date (ISO format) | "2023-05-15" |
| `due_date` | Payment due date (ISO format) | "2023-06-15" |
| `vendor` | Vendor information (object) | See below |
| `customer` | Customer information (object) | See below |
| `items` | Array of line items | See below |
| `subtotal` | Subtotal amount | 2000.00 |
| `tax` | Tax amount | 200.00 |
| `total` | Total amount | 2200.00 |
| `currency` | Currency code | "USD" |
| `payment_terms` | Payment terms | "Net 30" |
| `payment_instructions` | Instructions for payment | "Please pay by bank transfer..." |

Vendor and customer objects contain:

```json
{
  "name": "ACME Corporation",
  "address": "123 Main St, Anytown, USA",
  "phone": "+1 555-123-4567",
  "email": "billing@acme.com",
  "tax_id": "12-3456789"
}
```

### Use Cases

- Accounts payable automation
- Financial record keeping
- Supplier management
- Audit preparation

## ID Card Extraction

The `id_card` extraction type extracts information from government-issued ID cards, driver's licenses, and passports.

<Frame>
  <img src="/images/id-card-extraction.png" alt="ID Card Extraction Example" />
</Frame>

### Extracted Fields

| Field | Description | Example |
| ----- | ----------- | ------- |
| `document_type` | Type of ID document | "driver_license" |
| `id_number` | ID card number | "X12345678" |
| `full_name` | Full name of the ID holder | "John A. Smith" |
| `first_name` | First name | "John" |
| `last_name` | Last name | "Smith" |
| `date_of_birth` | Date of birth (ISO format) | "1985-07-15" |
| `issue_date` | Date of issue (ISO format) | "2020-01-10" |
| `expiry_date` | Expiration date (ISO format) | "2028-01-10" |
| `issuing_authority` | Authority that issued the ID | "Department of Motor Vehicles" |
| `nationality` | Nationality or country | "USA" |
| `address` | Address of the ID holder | "123 Main St, Anytown, CA 12345" |
| `gender` | Gender | "M" |

### Use Cases

- Customer onboarding (KYC)
- Identity verification
- Age verification
- Visitor management

## Business Card Extraction

The `business_card` extraction type extracts contact information from business cards.

<Frame>
  <img src="/images/business-card-extraction.png" alt="Business Card Extraction Example" />
</Frame>

### Extracted Fields

| Field | Description | Example |
| ----- | ----------- | ------- |
| `full_name` | Full name | "Jane B. Doe" |
| `job_title` | Job title or position | "Senior Marketing Manager" |
| `company` | Company or organization name | "XYZ Corporation" |
| `email` | Email address | "jane.doe@xyz.com" |
| `phone` | Primary phone number | "+1 555-123-4567" |
| `mobile` | Mobile phone number | "+1 555-987-6543" |
| `fax` | Fax number | "+1 555-789-0123" |
| `website` | Website URL | "www.xyz.com" |
| `address` | Physical address | "456 Business Ave, Suite 100, Metropolis, NY 10001" |
| `social_media` | Array of social media profiles | See below |

Social media profiles:

```json
[
  {
    "platform": "LinkedIn",
    "handle": "janedoe"
  },
  {
    "platform": "Twitter",
    "handle": "@jane_doe"
  }
]
```

### Use Cases

- Contact management
- CRM data entry
- Networking event follow-up
- Lead generation

## Document Extraction

The `document` extraction type is a general-purpose extractor for text documents, forms, and contracts.

<Frame>
  <img src="/images/document-extraction.png" alt="Document Extraction Example" />
</Frame>

### Extracted Data

The document extraction returns a structured representation of the document, including:

- Full text content
- Document structure (sections, paragraphs)
- Key-value pairs
- Tables (when present)
- Form fields (when present)

### Use Cases

- Contract analysis
- Form processing
- Document classification
- Text mining and analysis

## Table Extraction

The `table` extraction type is specialized for extracting tabular data from images and documents.

<Frame>
  <img src="/images/table-extraction.png" alt="Table Extraction Example" />
</Frame>

### Extracted Data

The table extraction returns:

```json
{
  "tables": [
    {
      "rows": 5,
      "columns": 3,
      "headers": ["Product", "Quantity", "Price"],
      "data": [
        ["Product A", "10", "$5.99"],
        ["Product B", "5", "$12.50"],
        ["Product C", "2", "$25.00"],
        ["Product D", "1", "$99.99"]
      ]
    }
  ]
}
```

### Use Cases

- Data entry automation
- Financial statement analysis
- Research data collection
- Report generation

## Custom Models

In addition to the standard extraction types, VizAPI.ai allows you to create custom extraction models tailored to your specific document formats and data needs.

Custom models can be trained on your own documents to extract exactly the fields you need with higher accuracy for your specific use case.

Learn more about [custom extraction models](/essentials/custom-models) in our documentation.
