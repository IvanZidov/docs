### Step-by-Step Creation

<Steps>
  <Step title="Start Building">
    Click "New Extraction" from your dashboard or "Create Custom Template" from the library.
  </Step>
  <Step title="Name & Describe">
    Give it a descriptive name and category. Add tags for better organization.
    
    <Note>Good names: "Invoice Data Extractor", "ID Card Processor", "Receipt Analyzer"</Note>
  </Step>
  <Step title="Define Fields">
    Add extraction fields using our comprehensive field types:
    
    <AccordionGroup>
      <Accordion title="Basic Fields" icon="text">
        - **String**: Names, descriptions, text content
        - **Number**: Amounts, quantities, IDs, measurements
        - **Date**: Timestamps, due dates, creation dates
        - **Boolean**: Yes/no, true/false, checkboxes
      </Accordion>
      <Accordion title="Advanced Fields" icon="layers">
        - **Enum**: Predefined lists (statuses, categories, types)
        - **Object**: Nested structures with subfields
        - **Image**: AI-generated images based on extracted context
      </Accordion>
    </AccordionGroup>
  </Step>
  <Step title="Configure Settings">
    Set up advanced options:
    - **Required fields**: Mark critical data as mandatory
    - **return_as_list**: Extract multiple instances
    - **Multi-language**: Support for 40+ languages
    - **Conditional logic**: Extract fields based on conditions
  </Step>
  <Step title="Test & Refine">
    Upload sample images to test your template. Monitor confidence scores and adjust field descriptions as needed.
  </Step>
  <Step title="Version & Deploy">
    Save your template as version 1.0 and make it available as an API endpoint.
  </Step>
</Steps>

## Field Types Explained

### Core Field Types

<Tabs>
  <Tab title="Text Fields">
    **Perfect for**: Names, labels, descriptions, addresses
    
    ```json
    {
      "name": "customer_name",
      "description": "The customer's full name as it appears on the document",
      "type": "string",
      "required": true,
      "return_as_list": false
    }
    ```
    
    **Best Practices**:
    - Be specific about location and format
    - Include context about expected content
    - Use clear, descriptive field names
  </Tab>
  <Tab title="Number Fields">
    **Perfect for**: Amounts, quantities, IDs, measurements
    
    ```json
    {
      "name": "total_amount",
      "description": "The total amount due, typically found at the bottom of the invoice",
      "type": "number",
      "required": true,
      "return_as_list": false
    }
    ```
    
    **Features**:
    - Automatic currency detection
    - Decimal precision handling
    - Negative number support
  </Tab>
  <Tab title="Date Fields">
    **Perfect for**: Dates, timestamps, time information
    
    ```json
    {
      "name": "invoice_date",
      "description": "The date the invoice was issued, usually in the header section",
      "type": "date",
      "required": false,
      "return_as_list": false
    }
    ```
    
    **Supports**:
    - Multiple date formats (MM/DD/YYYY, DD-MM-YYYY, etc.)
    - Relative dates ("yesterday", "last week")
    - International date formats
  </Tab>
  <Tab title="Complex Types">
    **Objects** for nested data:
    ```json
    {
      "name": "billing_address",
      "type": "object",
      "subfields": [
        {"name": "street", "type": "string"},
        {"name": "city", "type": "string"},
        {"name": "zip_code", "type": "string"}
      ]
    }
    ```
  </Tab>
</Tabs>

### Advanced Features

<AccordionGroup>
  <Accordion title="Conditional Fields" icon="branch">
    Extract fields only when certain conditions are met:
    
    ```json
    {
      "name": "sales_tax",
      "description": "Sales tax amount",
      "type": "number",
      "condition": "only_if_us_address",
      "required": false
    }
    ```
    
    **Use Cases**:
    - Tax fields for specific regions
    - Optional sections based on document type
    - Dynamic field extraction
  </Accordion>
  
  <Accordion title="Multi-language Support" icon="globe">
    Templates can handle documents in different languages:
    
    ```json
    {
      "name": "description",
      "description": "Product description",
      "type": "string",
      "language_hints": ["en", "es", "fr"],
      "target_language": "en"
    }
    ```
    
    **Features**:
    - Support for 40+ languages
    - Automatic language detection
    - Translation to target language
  </Accordion>
  
  <Accordion title="Confidence Thresholds" icon="gauge">
    Set minimum confidence requirements for field extraction:
    
    ```json
    {
      "name": "account_number",
      "type": "string",
      "required": true,
      "min_confidence": 0.85
    }
    ```
    
    **Benefits**:
    - Quality assurance
    - Reduced false positives
    - Automated validation
  </Accordion>
</AccordionGroup>

### Version Management Workflow

<Steps>
  <Step title="Create New Version">
    When editing a template, choose "Save as New Version" instead of overwriting.
    
    <Note>Version numbers are automatically incremented (1.0, 1.1, 2.0, etc.)</Note>
  </Step>
  <Step title="Test Thoroughly">
    Validate the new version with diverse sample images before deployment.
    
    **Test Checklist**:
    - High-quality images
    - Edge cases and variations
    - Different document formats
    - Multi-language content (if applicable)
  </Step>
  <Step title="Deploy Gradually">
    Make the new version active when you're confident it performs well.
    
    **Deployment Options**:
    - Set as default version
    - A/B test with traffic splitting
    - Gradual rollout schedule
  </Step>
  <Step title="Monitor Performance">
    Watch confidence scores, success rates, and user feedback after deployment.
    
    **Key Metrics**:
    - Average confidence scores
    - Field extraction success rate
    - API error rates
    - Processing time
  </Step>
</Steps>

<CodeGroup>

const response = await fetch('https://api.vizapi.ai/v1/agents/extract-values', {

## Common Issues & Solutions

<AccordionGroup>
  <Accordion title="Low Confidence Scores" icon="chart-line-down">
    **Problem**: AI isn't confident in extractions (scores below 0.8)
    
    **Solutions**:
    - **Improve descriptions**: Add more detail about field location and format
    - **Test image quality**: Use higher resolution, better lighting
    - **Field specificity**: Be more specific about expected data format
    - **Make optional**: Convert uncertain fields from required to optional
    
    **Example Fix**:
    ```json
    // Before (vague)
    {"name": "date", "description": "The date"}
    
    // After (specific)
    {"name": "invoice_date", "description": "The invoice date in MM/DD/YYYY format, typically found in the top-right corner of the document"}
    ```
  </Accordion>
  
  <Accordion title="Missing Data" icon="exclamation-triangle">
    **Problem**: Some fields consistently return empty or null
    
    **Solutions**:
    - **Verify presence**: Check if information is actually visible in test images
    - **Location hints**: Add specific location descriptions
    - **Alternative names**: Include synonyms or alternative field names
    - **Make optional**: Consider if field should be required
    
    **Debugging Process**:
    1. Upload the same image multiple times
    2. Check if field appears in any results
    3. Verify field description accuracy
    4. Test with different image quality/format
  </Accordion>
  
  <Accordion title="Wrong Data Types" icon="warning">
    **Problem**: Numbers returned as text, dates as strings, etc.
    
    **Solutions**:
    - **Verify field type**: Ensure type matches expected data format
    - **Format specifications**: Add format hints for dates and numbers
    - **Test edge cases**: Validate with diverse content formats
    - **Use enums**: For predictable values, use enum type
    
    **Type-Specific Tips**:
    - **Numbers**: Specify currency, decimal places, negative values
    - **Dates**: Mention expected format (MM/DD/YYYY vs DD/MM/YYYY)
    - **Booleans**: Clarify what represents true/false
  </Accordion>
  
  <Accordion title="Inconsistent Results" icon="shuffle">
    **Problem**: Same image gives different results across API calls
    
    **Solutions**:
    - **More specific descriptions**: Use detailed, unambiguous field descriptions
    - **Test variations**: Validate with multiple versions of similar content
    - **Confidence thresholds**: Set minimum confidence requirements
    - **Field simplification**: Split complex fields into simpler components
    
    **Consistency Checklist**:
    - [ ] Field descriptions are specific and unambiguous
    - [ ] Test images represent real-world variations  
    - [ ] Confidence scores are consistently high (>0.85)
    - [ ] Required fields are truly always present
  </Accordion>
</AccordionGroup>

### Template Design Guidelines

1. **Descriptive Names**: Use clear, specific field names (`invoice_total` vs `total`)
2. **Detailed Descriptions**: Include location, format, and context information
3. **Appropriate Types**: Match field types to expected data formats
4. **Smart Defaults**: Make fields optional unless absolutely required
5. **Version Management**: Use semantic versioning (1.0, 1.1, 2.0)
