## Field Types Explained

### Common Field Types

<Tabs>
  <Tab title="Text Fields">
    **Perfect for**: Names, labels, descriptions
    
    ```json
    {
      "name": "text_content",
      "description": "The main text content in the image",
      "type": "string",
      "required": true
    }
    ```
  </Tab>
  <Tab title="Number Fields">
    **Perfect for**: Amounts, quantities, numeric values
    
    ```json
    {
      "name": "numeric_value",
      "description": "A numerical value from the content",
      "type": "number",
      "required": true
    }
    ```
  </Tab>
  <Tab title="Date Fields">
    **Perfect for**: Dates, timestamps, time information
    
    ```json
    {
      "name": "date_field",
      "description": "A date found in the content",
      "type": "date",
      "required": false
    }
    ```
  </Tab>
  <Tab title="Lists">
    **Perfect for**: Multiple items, repeated entries
    
    ```json
    {
      "name": "list_items",
      "description": "Multiple similar items in the content",
      "type": "array",
      "required": false
    }
    ```
  </Tab>
</Tabs>

### Pro Tips for Better Extraction

<AccordionGroup>
  <Accordion title="Write Clear Descriptions" icon="pencil">
    ❌ Poor: "Get the date"
    
    ✅ Good: "Extract the date information, typically found in the header area"
  </Accordion>
  
  <Accordion title="Be Specific" icon="target">
    Include context about where information is typically located and what format to expect.
  </Accordion>
  
  <Accordion title="Mark Required Carefully" icon="exclamation">
    Only mark fields as required if they're always present in your content.
  </Accordion>
  
  <Accordion title="Test with Real Data" icon="flask">
    Use actual images from your use case, not just sample content.
  </Accordion>
</AccordionGroup>

## Using Templates via API

### Making Extraction Calls

Once you have a template, using it is simple:

<CodeGroup>

```javascript JavaScript
const response = await fetch('https://api.vizapi.ai/v1/agents/extract-values', {
  method: 'POST',
  headers: {
    'x-api-key': 'YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    template_id: 'your-template-id',
    image_url: 'https://example.com/document.jpg'
  })
});

const result = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.vizapi.ai/v1/agents/extract-values',
    headers={
        'x-api-key': 'YOUR_API_KEY',
        'Content-Type': 'application/json'
    },
    json={
        'template_id': 'your-template-id',
        'image_url': 'https://example.com/document.jpg'
    }
)

result = response.json()
```

```bash cURL
curl -X POST https://api.vizapi.ai/v1/agents/extract-values \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "template_id": "your-template-id",
    "image_url": "https://example.com/document.jpg"
  }'
```

</CodeGroup>

## Common Issues & Solutions

<AccordionGroup>
  <Accordion title="Low Confidence Scores" icon="chart-line-down">
    **Problem**: AI isn't confident in extractions
    
    **Solutions**:
    - Improve field descriptions with more detail
    - Test with higher quality images
    - Make uncertain fields optional instead of required
  </Accordion>
  
  <Accordion title="Missing Data" icon="exclamation-triangle">
    **Problem**: Some fields consistently return empty
    
    **Solutions**:
    - Check if the information is actually visible in images
    - Improve field descriptions to help AI locate data
    - Make fields optional if they're not always present
  </Accordion>
  
  <Accordion title="Wrong Data Types" icon="warning">
    **Problem**: Numbers coming back as text, dates as strings
    
    **Solutions**:
    - Verify field type matches expected data format
    - Add format specifications for dates and numbers
    - Test with diverse content formats
  </Accordion>
  
  <Accordion title="Inconsistent Results" icon="shuffle">
    **Problem**: Same image gives different results
    
    **Solutions**:
    - Use more specific field descriptions
    - Test with multiple versions of similar content
    - Consider splitting complex fields into simpler ones
  </Accordion>
</AccordionGroup>

## Advanced Features

### Multi-Language Support

Templates can handle documents in different languages:

```json
{
  "name": "total_amount",
  "description": "Total amount to pay",
  "type": "number",
  "language_hints": ["en", "es", "fr"]
}
```

### Conditional Fields

Extract fields only when certain conditions are met:

```json
{
  "name": "sales_tax",
  "description": "Sales tax amount",
  "type": "number",
  "condition": "only_if_us_address"
}
```

### Batch Processing

Process multiple images with one API call:

```json
{
  "template_id": "data-extractor",
  "images": [
    "https://example.com/image1.jpg",
    "https://example.com/image2.jpg",
    "https://example.com/image3.jpg"
  ]
}
```
